<objective>
Extend the IPC bridge so the renderer can request a window resize by sending {width, height} to the main process, which calls win.setSize() and repositions the window near the tray icon.

Purpose: The settings panel toggle needs to grow/shrink the Electron window. This cannot happen from renderer-side code — it must go through IPC to the main process. This plan establishes that channel before any UI is wired.

Output: preload.js exposes window.glorb.resize(w, h); main.js handles the resize-window channel and repositions the window after resizing.
</objective>

<task type="auto">
  <name>Task 2: Handle resize-window IPC in main.js</name>
  <files>main.js</files>
  <read_first>main.js</read_first>
  <action>
Add an ipcMain.handle for 'resize-window' at the bottom of main.js, after the existing 'quit-app' handler. The handler must:

1. Receive the payload `{ width, height }` from the second argument (event, args)
2. Call `win.setSize(width, height)` — integers required, use Math.round() on both values
3. Reposition the window so it stays anchored near the tray icon center. Use the same positioning formula as `createTray()`, but adjust for the new width:

```js
ipcMain.handle('resize-window', (event, { width, height }) => {
  win.setSize(Math.round(width), Math.round(height))
  const bounds = tray.getBounds()
  win.setPosition(
    Math.round(bounds.x + bounds.width / 2 - width / 2),
    Math.round(bounds.y + bounds.height)
  )
})
```

The existing `createTray` click handler positions using `-143` (half of 286px). The new handler uses `-width/2` so it works for both 286px (collapsed) and 440px (expanded).

No other changes to main.js. The existing quit-app handler and all other code remain untouched.
