## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, hamburger bar internal spacing |
| sm | 8px | Hamburger button margin from window edge (top-right 8px), close button position (carried from Phase 1) |
| md | 16px | Window inner horizontal padding, button side padding |
| lg | 24px | Button side padding (full-width button: `padding-left: 24px; padding-right: 24px`), vertical gap between ring and time display |
| xl | 32px | Vertical gap between time display and Start button |
| 2xl | 48px | Top padding from window top edge to ring top |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- Ring diameter: 200px (70% of 286px window width — set by CONTEXT.md decision, not a spacing token)
- Ring stroke width: 8px (SVG property, not a spacing token)
- glorb image: 130px diameter (65% of 184px ring interior — CONTEXT.md decision)
- Hamburger button touch target: 28×28px (matches × close button established in Phase 1)
- Time display vertical padding: 8px top / 8px bottom (compact label treatment)

Source: 02-CONTEXT.md (ring 200px, stroke 8px, 65% interior, 24px side padding, 8px hamburger margin), 01-UI-SPEC.md (md=16px, sm=8px)

## Component Inventory

### 1. SVG Ring Timer

- Container: `<div id="timer-ring">`, centered horizontally, `margin: 0 auto`
- SVG element: `width="200" height="200"` `viewBox="0 0 200 200"`
- Center coordinates: `cx="100" cy="100"`
- Outer radius: `r="92"` (200px diameter − 8px stroke × 2 = 184px interior; radius = 92px)
- `stroke-linecap: round` — softens the shrinking arc tip for a polished feel
- Circumference: `2 × π × 92 ≈ 578px` — declare as JS constant `CIRCUMFERENCE = 2 * Math.PI * 92`

Track circle (static background):
- `stroke: rgba(26, 26, 26, 0.1)`
- `stroke-width: 8`
- `fill: none`
- `stroke-dasharray: none`

Progress circle (animated foreground):
- `id="ring-progress"`
- `stroke: #FF6B35`
- `stroke-width: 8`
- `fill: none`
- `transform: rotate(-90deg)` with `transform-origin: center` — sets 12-o'clock start
- `stroke-dasharray: {CIRCUMFERENCE}`
- `stroke-dashoffset: 0` at idle (full ring = 100% session available)
- `stroke-dashoffset` decreases toward `CIRCUMFERENCE` as time passes (clockwise depletion)
- CSS transition: `stroke-dashoffset 1s linear` — smooth per-tick animation

Source: 02-CONTEXT.md (200px diameter, 8px stroke, rgba track, clockwise depletion, idle = full orange, stroke-dashoffset), REQUIREMENTS.md TIMER-01

### 2. glorb.png Mascot

- Container: `<div id="glorb-container">`, absolutely positioned at SVG center
- Size: 130×130px (`width: 130px; height: 130px` — 65% of 200px ring diameter per CONTEXT.md)
- Position: `position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%)`
- Clip: `border-radius: 50%; overflow: hidden`
- Image element: `<img id="glorb-img" src="assets/glorb.png" alt="Glorb mascot">`
- Image fills container: `width: 100%; height: 100%; object-fit: cover`
- Vignette overlay: `<div id="glorb-vignette">` — `position: absolute; inset: 0; border-radius: 50%; background: radial-gradient(circle, transparent 40%, #f0f0f0 75%); pointer-events: none`

Placeholder (when glorb.png absent or fails to load):
- Hide `<img>` via `display: none` on error
- Show `<div id="glorb-placeholder">`: `background: #FF6B35; border-radius: 50%; width: 130px; height: 130px; display: flex; align-items: center; justify-content: center`
- Placeholder text: "G" — `font-size: 36px; font-weight: 600; color: #ffffff; font-family: Inter, system-ui, sans-serif`

Source: 02-CONTEXT.md (65% interior, circular clip, radial-gradient vignette to #f0f0f0, orange placeholder with "G"), REQUIREMENTS.md TIMER-02

### 3. Time Display

- Element: `<div id="time-display">`
- Position in layout: below ring, above Start button
- Format: "XXh XXm" (e.g. "00h 25m" at idle, "00h 24m" after first minute) — TIMER-03
- Typography: Display role — 28px, weight 600, #1a1a1a, line-height 1.2
- Text-align: center
- Margin: `margin-top: 24px` (lg token) from ring bottom

Source: REQUIREMENTS.md TIMER-03 ("XXh XXm" format), 02-CONTEXT.md (placement: ring → time → button), 01-UI-SPEC.md (Display typography)

### 4. Start / Pause Button

- Element: `<button id="btn-start">`
- Width: full-width with `padding-left: 24px; padding-right: 24px` (lg token sides), `padding-top: 12px; padding-bottom: 12px`
- Margin: `margin-top: 32px` (xl token) from time display
- Typography: 14px, weight 400, #ffffff
- Background: #FF6B35 (filled accent — primary action)
- Border: `1px solid #FF6B35`
- Border-radius: 8px (consistent with Phase 1 overlay buttons)
- Hover: `background: #e55a26; border-color: #e55a26`
- Transition: `background 150ms ease, border-color 150ms ease`
- Default label: "Start" (idle state)
- Running label: "Pause"
- Paused label: "Resume"
- Action: toggles timer running/paused state; updates own label

Source: 02-CONTEXT.md (filled #FF6B35, full-width 24px padding, idle = full orange ring → button says "Start"), REQUIREMENTS.md TIMER-04 TIMER-06

### 5. Hamburger Button (Phase 3 placeholder)

- Element: `<button id="btn-hamburger">`
- Position: `position: absolute; top: 8px; right: 8px` (mirrors × close button at top-left)
- Size: 28×28px touch target (consistent with × close button)
- Background: transparent
- Border: none
- Border-radius: 6px
- Hover: `background: rgba(26,26,26,0.08)` — neutral hover, not accent
- Transition: `background 150ms ease`
- Content: 3 horizontal bars (`<span>` elements or inline SVG)

Hamburger bars (3 CSS spans stacked):
- Each bar: `width: 14px; height: 2px; background: #1a1a1a; border-radius: 1px; display: block`
- Vertical gap between bars: 3px
- All 3 bars wrapped in a `<div>` flex column with `gap: 3px`, centered inside button

- Action (Phase 2): no-op — button is placed but not wired until Phase 3
- aria-label: "Open settings"
- aria-disabled: "true" in Phase 2 (wired in Phase 3)

Source: 02-CONTEXT.md (hamburger top-right 8px margin, mirrors × close), REQUIREMENTS.md SETT-01 (Phase 3 dependency, placed here)

### 6. Timer Layout Container

- Element: `<div id="timer-view">`, child of `#app`
- Position: fills `#app` entirely: `width: 100%; height: 100%; position: relative`
- Layout: vertical flex column, `display: flex; flex-direction: column; align-items: center`
- Padding: `padding: 48px 16px 32px 16px` (top 48px = 2xl token, sides 16px = md token, bottom 32px = xl token)
- This container holds: ring → time display → button (top-to-bottom order)
- The × close button (`#btn-close`) and hamburger button (`#btn-hamburger`) remain absolutely positioned over this container

Source: 02-CONTEXT.md (top-to-bottom: ring → time → start), 01-UI-SPEC.md (md=16px side padding), CONTEXT.md vertical spacing is Claude's discretion

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (idle) | "Start" |
| Primary CTA (running) | "Pause" |
| Primary CTA (paused) | "Resume" |
| Time display (idle) | "00h 25m" |
| Time display (running) | "XXh XXm" (live countdown) |
| Time display (complete) | "00h 00m" — resets to "00h 25m" immediately |
| glorb placeholder alt | "Glorb mascot" |
| glorb placeholder label | "G" (single letter inside orange circle placeholder) |
| Hamburger button label | aria-label: "Open settings" |
| Empty state | not applicable — timer is always in a defined state (idle/running/paused) |
| Error state | not applicable — Phase 2 has no async operations; timer failure is silent reset |
| Destructive actions | none new in Phase 2 — quit overlay carried from Phase 1 unchanged |

Button label rationale: "Start" → "Pause" → "Resume" (not "Start" again after first pause) communicates that the session is in progress. "Resume" is clearer than "Start" when time has elapsed.

Source: REQUIREMENTS.md TIMER-03 ("XXh XXm" format), TIMER-06 (button label toggles), 02-CONTEXT.md (idle = full ring → implies "Start"), CLAUDE.md (minimalist — no extra copy)

## Interaction Contract

| Trigger | Result | Notes |
|---------|--------|-------|
| Window opens (idle) | Ring at 100% orange stroke; time shows "00h 25m"; button says "Start" | CONTEXT.md idle state decision |
| Click "Start" | Timer begins 1s tick; ring depletes clockwise; button changes to "Pause" | TIMER-05, TIMER-06 |
| Click "Pause" (running) | Timer freezes; ring holds current position; button changes to "Resume" | TIMER-06 |
| Click "Resume" (paused) | Timer continues from frozen position; button changes to "Pause" | TIMER-06 |
| Timer reaches 00:00 | Ring empties fully then instantly resets to 100% orange; time resets to "00h 25m"; button resets to "Start" | TIMER-07 |
| glorb.png load fails | `onerror` hides `<img>`, shows orange placeholder div with "G" | TIMER-02 / CONTEXT.md placeholder |
| Click hamburger | No-op in Phase 2 (button present, not wired) | Phase 3 dependency |
| Click × close | Quit confirmation overlay (unchanged from Phase 1) | Phase 1 contract |
| Blur window | Window hides (unchanged from Phase 1) | Phase 1 contract |

Ring animation: `stroke-dashoffset` is set each tick via `setInterval(tick, 1000)`. CSS `transition: stroke-dashoffset 1s linear` handles the interpolation between ticks — no requestAnimationFrame required for v1. At pause, remove the CSS transition temporarily to freeze position without animation drift; restore on resume.

Source: 02-CONTEXT.md (1s setInterval, stroke-dashoffset animation, CONTEXT.md discretion for timing function), REQUIREMENTS.md TIMER-05 TIMER-06 TIMER-07
