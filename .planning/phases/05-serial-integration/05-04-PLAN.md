  <action>
Compile and upload the firmware to the connected Arduino UNO:

```bash
cd /Users/ouen/slop/glorb/firmware && pio run --target upload
```

If multiple serial ports are detected and PlatformIO cannot auto-select, it will prompt for the port. In that case, identify the Arduino port first:
```bash
ls /dev/cu.usb* /dev/cu.wchusbserial* 2>/dev/null
```
Then upload with explicit port:
```bash
pio run --target upload --upload-port /dev/cu.usbmodemXXXX
```

Wait for the upload to complete. The Arduino will reset and start running the new firmware. Both displays should show the SMILE state (closed eyes arc) immediately on power-up, per D-08.

If `pio` is not in PATH, use the full path: `~/.platformio/penv/bin/pio` or `platformio`.

<task type="auto">
  <name>Task 2: Test serial commands via serial monitor</name>
  <files></files>
  <read_first>
    - firmware/platformio.ini — confirm monitor_speed = 115200
  </read_first>
  <action>
Use PlatformIO serial monitor to send commands and verify display state changes.

Open monitor (Ctrl+C to exit):
```bash
cd /Users/ouen/slop/glorb/firmware && pio device monitor --baud 115200
```

In the serial monitor, type and send each command (press Enter after each):
1. `DEFAULT` — both displays should switch to OPEN_EYES (hollow ring ellipses)
2. `SMILE` — both displays should switch to SMILE (upper arc)
3. `DEFAULT` again — confirm toggle back

If the serial monitor is interactive, type the string and press Enter. The firmware uses `Serial.readStringUntil('\n')` so the newline from Enter terminates the command.

Alternative: if monitor is not interactive, use `echo` with the port directly:
```bash
PORT=$(ls /dev/cu.usb* /dev/cu.wchusbserial* 2>/dev/null | head -1)
echo "DEFAULT" > $PORT
sleep 1
echo "SMILE" > $PORT
```

Visually confirm: DEFAULT = two hollow oval rings on each display; SMILE = two upward-curving arcs on each display.
  </action>
  <verify>
    Visual hardware verification — both displays respond to commands
  </verify>
  <acceptance_criteria>
    - After sending "DEFAULT": both OLED displays show a hollow oval ring (outline ellipse, dark center)
    - After sending "SMILE": both OLED displays show an upward-curving arc (upper ellipse arc, no font character)
    - Commands toggle state correctly (can send DEFAULT after SMILE and back)
    - Unknown input (e.g. blank enter, garbage text) does not crash or corrupt display
  </acceptance_criteria>
  <done>Both displays respond to DEFAULT and SMILE commands sent over USB serial with the correct visual states.</done>
</task>
