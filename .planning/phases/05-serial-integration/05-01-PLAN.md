Rewrite firmware/src/main.cpp as follows:

```cpp
#include <Arduino.h>
#include "display.h"

void setup() {
    Serial.begin(115200);
    displaySetup();
    showDisplay(DisplayState::SMILE);  // D-08: power-up default = SMILE (closed eyes)
}

void loop() {
    // D-01: blocking readStringUntil — loop does nothing else, blocking is fine
    String cmd = Serial.readStringUntil('\n');
    cmd.trim();

    // D-03: exact string match after trim
    if (cmd == "DEFAULT") {
        showDisplay(DisplayState::OPEN_EYES);
    } else if (cmd == "SMILE") {
        showDisplay(DisplayState::SMILE);
    }
    // Unknown commands silently ignored
}
```

Key points:
- `readStringUntil('\n')` blocks until newline arrives or timeout (default 1000ms) — this is intentional (D-01)
- `cmd.trim()` handles "\r\n" line endings from any sender
- `setup()` calls `showDisplay(DisplayState::SMILE)` not OPEN_EYES (D-08: closed eyes on power-up)
- The cycling `delay()` demo loop is completely removed (D-02)
- No changes to platformio.ini (baud 115200 already set via monitor_speed, Serial.begin is in code)
