## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label-to-dot gap in response selector |
| sm | 8px | Inline button padding, tight element spacing |
| md | 16px | Default element spacing, horizontal content padding |
| lg | 24px | Between question text and response selector |
| xl | 32px | Between response selector and navigation row |
| 2xl | 48px | Vertical centering buffer, section breaks |
| 3xl | 64px | Top/bottom padding on greeting and completion screens |

Exceptions:
- Response dots: 48px diameter (touch-target minimum, matches CONTEXT.md spec)
- Dot gap: 20px between dot centers — gives visual breathing room across 800px width
- Progress bar height: 3px — intentionally sub-scale for visual lightness
- Back button disabled opacity: 0.35 — below default disabled 0.5 to signal "hard disabled"

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Name input placeholder, helper text, button labels |
| Label | 14px | 400 | 1.5 | "Question X of 18" counter, "Never" / "Very Often" endpoint labels, Back button text |
| Heading | 20px | 600 | 1.3 | Question text, name prompt "What should I call you?" |
| Display | 28px | 600 | 1.2 | Greeting headline "Hi, I'm Glorbus Pallidus. You can call me Glorb.", completion message |

All sizes step up from renderer.css base (14px body → 16px for onboarding because the window is 2.8× wider and text must breathe at that scale).

Font stack matches renderer.css exactly: `Inter, system-ui, -apple-system, BlinkMacSystemFont, sans-serif`.

1. Progress bar fill (top of questionnaire screens)

Source: CLAUDE.md design spec, renderer.css tokens.

## Animation

### Breathing Gradient (Greeting Screen Only)

| Property | Spec |
|----------|------|
| Type | CSS `@keyframes` on a pseudo-element or dedicated div behind content |
| Shape | Radial gradient centered on Glorb image position |
| Light mode | `radial-gradient(circle at center, #FF6B35 0%, #f0f0f0 60%)` |
| Dark mode | `radial-gradient(circle at center, #FF6B35 0%, #171719 60%)` |
| Keyframe | `0%` → `transform: scale(0.85); opacity: 0.55` → `50%` → `transform: scale(1.15); opacity: 0.80` → `100%` → `transform: scale(0.85); opacity: 0.55` |
| Duration | 3.6s |
| Easing | `ease-in-out` |
| Iteration | `infinite` |
| Layer | Behind all content (`z-index: 0`); content on `z-index: 1` |
| Size | Cover full window (position absolute, inset 0) |
| Reduce motion | `@media (prefers-reduced-motion: reduce)` — disable animation, show static gradient at scale(1) opacity 0.6 |

### Screen Transitions (Questionnaire Navigation)

| Property | Spec |
|----------|------|
| Forward advance | Content fades out (`opacity 0 → 1`) over 120ms; new content fades in over 120ms |
| Back navigation | Same 120ms fade — no directional slide (keeps it simple for vanilla CSS) |
| Auto-advance | 250ms delay after dot selection, then 120ms fade transition |
| Implementation | Toggle `.screen-exit` / `.screen-enter` CSS classes on the content wrapper |

### General Transitions

All interactive elements inherit the 150ms ease transition from renderer.css conventions.

## Accessibility Notes

- Response dots must have `role="radio"` and `aria-checked` toggled on selection
- Dot group wrapped in `role="radiogroup"` with `aria-label` set to the question text
- Back button uses `disabled` attribute (not just visual styling) on Q1
- Progress bar: `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="18"`
- Name input: `aria-label="Your name"` or associated `<label>` element
- Keyboard: Tab moves between dots; Space/Enter selects focused dot; Enter on name input advances
- Focus ring: do not suppress `outline` on keyboard-focused dots — use `:focus-visible` only
