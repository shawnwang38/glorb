# Phase quick Plan 260417-fvf: Replace drawEllipse with XBM bitmaps Summary

**One-liner:** XBM byte arrays for hollow ellipse ring (OPEN_EYES) and upper arc (SMILE) computed via Python, stored in PROGMEM bitmaps.h, rendered via U8g2 drawXBM replacing all programmatic draw calls.

## What Was Done

Replaced the hardcoded U8g2 drawEllipse math in display.cpp with XBM bitmap rendering so graphics are user-swappable by editing a single header file.

### Task 1: Create bitmaps.h

Created `firmware/src/bitmaps.h` with two 1024-byte PROGMEM arrays:

- **open_eyes_bits**: Hollow ellipse ring, center (32,64), rx=20, ry=30. Generated by checking each pixel (x,y) for `abs((dx^2/400) + (dy^2/900) - 1) < 0.12`. Mirrors the previous `drawEllipse(32, 64, 20, 30, U8G2_DRAW_ALL)` call.
- **smile_bits**: Upper arc only, center (32,90), rx=20, ry=30, pixels at y>=90 always zero. Mirrors the previous `drawEllipse(32, 72, 20, 10, U8G2_DRAW_UPPER_LEFT | U8G2_DRAW_UPPER_RIGHT)` call (plan revised center/radii to match the symmetrical ellipse approach).

Arrays were generated by running the Python reference script from the plan constraints. Format: LSB-first XBM, 8 bytes per row, 128 rows = 1024 bytes per image.

### Task 2: Update display.cpp

- Added `#include "bitmaps.h"` after `#include "display.h"`
- Removed `drawEyePage` and `drawSmilePage` static functions entirely
- Removed `typedef void (*DrawFn)(U8G2 &)` indirection pattern
- Rewrote `showDisplay` to select `open_eyes_bits` or `smile_bits` and call `u8g2_hw.drawXBM(0, 0, 64, 128, bmp)` and `u8g2_sw.drawXBM(0, 0, 64, 128, bmp)`
