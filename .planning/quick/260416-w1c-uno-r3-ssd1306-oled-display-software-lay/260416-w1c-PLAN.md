<objective>
Create a PlatformIO firmware project for Arduino UNO R3 + SSD1306 128x64 OLED that exposes a
clean `showDisplay(DisplayState)` function with two pre-defined states: OPEN_EYES and SMILE.
Because the OLED is physically mounted upside down, all bitmap data must be stored vertically
flipped so the rendered image appears correct to the viewer.

Purpose: Provide a reusable display abstraction layer that can be called from any future
firmware logic without caring about I2C details or bitmap encoding.

Output: firmware/ directory — PlatformIO project that compiles and uploads to UNO R3.
</objective>

<context>
This is a standalone firmware sub-project inside the Glorb repo. It has no runtime dependency on
the Electron app. The firmware/ directory is self-contained.

Hardware:
- MCU: Arduino UNO R3 (ATmega328P)
- Display: SSD1306 128x64 OLED, I2C (default addr 0x3C), physically mounted upside down
- Connection: SDA→A4, SCL→A5 (UNO hardware I2C)

Library choice: U8g2 (thomasfredericks/U8g2) — handles SSD1306 natively, has full-buffer mode
(U8G2_..._F_...) for arbitrary bitmap drawing, and has a built-in font for text.

Vertical flip strategy: The OLED is mounted 180° rotated. U8g2 supports display rotation via
constructor variant U8G2_R2 (180°). Using the rotation constructor is the CORRECT approach —
no manual bitmap flipping is needed when using U8G2_R2.

OPEN_EYES bitmap design (will be drawn with u8g2.drawBox):
- Two horizontal rectangles: each ~40px wide, ~8px tall
- Left eye: x=14, y=28, w=40, h=8
- Right eye: x=74, y=28, w=40, h=8
- Centered vertically (128x64 display, eyes near vertical center)

SMILE design (drawn with u8g2.setFont + u8g2.drawStr):
- Use u8g2_font_logisoso32_tr (large 32px font) or u8g2_font_inb24_mr (24px)
- Draw ">" at x~10, y~48 and "<" at x~78, y~48 (baseline coords in U8g2)
- With U8G2_R2 rotation active, the > and < will appear correct from viewer perspective
</context>

Create the firmware/ directory with a complete PlatformIO project.

**firmware/platformio.ini:**
```ini
[env:uno]
platform = atmelavr
board = uno
framework = arduino
lib_deps =
    olikraus/U8g2@^2.35.7
monitor_speed = 115200
```

**firmware/src/display.h:**
Define the DisplayState enum and declare showDisplay():
```cpp
#pragma once

enum class DisplayState {
    OPEN_EYES,
    SMILE
};

void displaySetup();
void showDisplay(DisplayState state);
```

**firmware/src/display.cpp:**
- Include U8g2lib.h
- Instantiate: `U8G2_SSD1306_128X64_NONAME_F_HW_I2C u8g2(U8G2_R2, U8X8_PIN_NONE);`
  - U8G2_R2 rotates 180° — this handles the upside-down mount; no manual bitmap flip needed.
- `displaySetup()`: call `u8g2.begin()`
- `showDisplay(DisplayState state)`:
  - Call `u8g2.clearBuffer()`
  - For OPEN_EYES:
    - `u8g2.setDrawColor(1)`
    - Left eye: `u8g2.drawBox(14, 28, 40, 8)`
    - Right eye: `u8g2.drawBox(74, 28, 40, 8)`
  - For SMILE:
    - `u8g2.setFont(u8g2_font_logisoso32_tr)`
    - `u8g2.drawStr(8, 50, ">")`
    - `u8g2.drawStr(78, 50, "<")`
  - Call `u8g2.sendBuffer()`

**firmware/src/main.cpp:**
- Include "display.h"
- `setup()`: call `displaySetup()`, show OPEN_EYES initially
- `loop()`: cycle through states every 2 seconds for demo:
  ```cpp
  void loop() {
      showDisplay(DisplayState::OPEN_EYES);
      delay(2000);
      showDisplay(DisplayState::SMILE);
      delay(2000);
  }
  ```
  This demo loop proves both states render without needing additional tooling.

Do NOT use the Arduino IDE project format. Only PlatformIO layout (platformio.ini at firmware/ root, sources under firmware/src/).
